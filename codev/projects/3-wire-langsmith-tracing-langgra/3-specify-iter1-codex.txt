The spec assumes `langsmith.evaluate()` works with only `LANGCHAIN_API_KEY`. In practice, you’ll need explicit LangSmith client setup and possibly `LANGCHAIN_ENDPOINT` or `LANGSMITH_ENDPOINT` depending on SDK version. The spec should pin the exact SDK version and required env vars. Right now, it only mandates `langsmith>=0.1` and lists `LANGCHAIN_ENDPOINT` but does not clarify compatibility or required variables for the SDK version. This can break the eval upload flow.

This ambiguity will affect dataset creation and evaluation.

3. CLI flag behavior conflicts with env-only toggle.
The spec says “use a feature flag `--langsmith`” but also says “if `LANGCHAIN_API_KEY` is set, upload.” It’s unclear which is authoritative. Define precedence clearly to avoid surprises.
